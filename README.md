# Fuzzy Translator

A simple multi-language translation API using the [Facebook NLLB](https://huggingface.co/facebook/nllb-200-distilled-600M) transformer from HuggingFace, with a [lightweight toxicity classifier](https://huggingface.co/s-nlp/roberta_toxicity_classifier) that screens inappropriate input.

**Supported languages:**

- 🇬🇧 English 
- 🇰🇷 Korean
- 🇨🇳 Chinese (Simplified)
- 🇹🇼 Chinese (Traditional)
- 🇪🇸 Spanish
- 🇬🇷 Greek

## Installation

**Prerequisites:** Python 3.11+

**NOTE:** the models will be downloaded on the first run. This usually takes a few minutes.

#### With Docker (recommended)

1. `docker build -t fuzzy_translator .`
2. `docker run -p 8000:8000 fuzzy_translator`

#### Without Docker

1. Create a fresh virtual environment (recommended)
2. `pip install -r requirements.txt`
3. `uvicorn app:app --reload`

## Project Structure

- `app.py`          - FastAPI application and translation endpoint

- `requirements.txt` - dependencies
- `Dockerfile`      - container setup

## API Reference

Interactive API docs (Swagger UI) can be found at **http://127.0.0.1:8000/docs**

#### `GET /ready`

Returns the readiness status of the API.

#### `GET /languages`

Lists the languages available for translation.

#### `GET /metrics`

Returns Prometheus-formatted metrics.

#### `POST /translate`

Translates text from source to target language.

**Request body:**
| Field    | Type   | Description                    |
|----------|--------|--------------------------------|
| `text`   | string | Text to translate (1-100 characters)|
| `source` | string | Source language                 |
| `target` | string | Target language                |

**Example usage:**

🗣️ `curl http://127.0.0.1:8000/translate -H "Content-Type: application/json" -d '{"text": "Hello!", "source": "english", "target": "korean"}'`

↩️ `{"translation": "안녕하세요!"}`

| Status Code | Description                                       |
|-------------|---------------------------------------------------|
| 200         | Translation successful                            |
| 422         | Validation error (input empty/exceeds 100 chars, unsupported language) |
| 503         | Models not loaded                                 |
| 500         | Unexpected server error                           |

## Design Decisions

**Translation model:** `facebook/nllb-200-distilled-600M` is a widely used multilingual translation model on HuggingFace. The distilled version is roughly half the size of the 1.3B parameter model, making it a more practical choice for this project.

**FastAPI:** chosen due to its simplicity, autogenerated documentation, Pydantic request validation, and integrated JSON error handling.

**Lifespan:** the models are preloaded at startup with FastAPI's lifespan hook. Loading can take a few minutes, so performing this on every request would be inefficient.

## Production Considerations

**Concurrency:** the API currently runs on uvicorn with a single worker. To handle multiple requests simultaneously, multiple container replicas could be deployed behind a load balancer, each holding its own copy of the models in memory.

**Monitoring:** `/metrics` currently exposes request latency, error rates, and system resource usage. These metrics could be scraped by a monitoring system such as Prometheus and visualised in a dashboard to monitor trends over time.

**Scaling:** each container replica needs its own copy of the models, so memory becomes a key constraint when scaling. For larger deployments, inference could be moved to a dedicated model-serving layer that handles translation requests across multiple API replicas.

**Testing:** unit tests use mocked models to keep tests fast and deterministic. In production, an integration test could run in CI on each push to ensure the full pipeline works correctly before allowing merges.

**Input validation:** enforced via Pydantic request validation and screened for inappropriate content using `s-nlp/roberta_toxicity_classifier`.

**User feedback:** a `/feedback` endpoint could collect reports of incorrect translations. These examples could then be reviewed and used to improve translation behaviour.

**Continuous evaluation:** translations could be periodically evaluated against a benchmark dataset, tracking metrics such as BLEU to detect performance drift over time.
